# 프로젝트 요약

## 개요

로컬 LLM(Ollama)을 나레이터로 사용하는 텍스트 RPG 웹 게임.
핵심 목표는 **정해진 스토리 라인이 아니라 플레이어가 만들어가는, 매번 다른 이야기**입니다.
고정된 것은 서장(프롤로그)과 상점뿐이며, 몬스터·전리품·이야기 전개는
AI가 플레이 맥락에 맞춰 동적으로 생성합니다.

## 구현된 기능

### 이야기 / AI
- 자유 텍스트 행동 입력 → AI 나레이터 서사 (NDJSON 스트리밍, 첫 글자 ~1초)
- 직업별 고정 프롤로그 → AI 초기 기억에 시드 → 선택에 따라 이야기 분기
- 3계층 기억: 최근 대화 10개(원문) + 증분 병합 요약 + 게임 상태
- 요약 유실 방지: pending 영속화 + 서버 재시작 시 자동 재시도 + 하드 상한(100)
- AI 몬스터 창작 (지역/이야기 반영, JSON 강제 모드, 실패 시 고전 DB 폴백)
- AI 전리품 창작 (이름/묘사는 AI, 스탯/가격은 레벨 공식)
- 지도 소지 시 무작위 퀘스트 훅 이벤트
- 새로고침 시 대화/요약 복원

### RPG 시스템
- 직업 5종 (전사/도적/마법사/성기사/레인저) — 스탯, 공격력 공식, 레벨업 성장 차별화
- 전투: 플레이어 레벨 ±(-1~+2) 적, 데미지 편차 ±10%, 민첩 기반 도주
- 보스전: 10% 확률 (레벨 3부터), 체력 1.8배/공격 1.2배, 보상 3배 + 장비 확정 드롭
- 사망 페널티: 골드 10% 손실 + 마을 부활
- 레벨/경험치 (직업별 성장), 골드 경제
- 인벤토리 20슬롯 (스택), 장비 장착/해제 — 가득 참 상황에서 아이템 소멸 방지 처리
- 상점: 구매/판매 (판매가 = 정가 절반, 가격 미상 아이템은 효과 기반 추정)
- 퀘스트 3종: 지역 소탕(3/5/8마리) / 보스 토벌 / 탐험(다른 지역 2곳) — 최대 5개, 완료 기록 20개
- 지역 5곳 이동, 마을 여관 휴식 (10+레벨×5 골드)
- 통계: 처치/사망/완료 퀘스트

### 안정성 / 운영
- 원자적 저장 + 백업 자동 복구 + 메모리 캐시
- AI 예외 분리 처리 (타임아웃/연결 실패별 로그 + 사용자 안내)
- 서버 시작 시 모델 프리워밍
- Ollama 미실행 시에도 게임 동작 (폴백)
- 환경변수: `OLLAMA_MODEL`, `OLLAMA_URL`, `TEXTRPG_AI_GEN`
- 구버전 세이브 자동 호환 (새 필드 기본값 로드)

### UI
- 좌측 채팅(이야기) + 우측 패널(스탯/장비/인벤토리/퀘스트/이동/기록)
- 검정 배경 + 흰 테두리, 이모지/이미지 없음, 전체 한국어
- 스트리밍 실시간 렌더링, 응답 대기 표시, 직업 선택/상점/퀘스트 게시판 모달

## 실행 환경

- Python 3.10+ / FastAPI / httpx / Pydantic
- Ollama + gemma2:2b (1.6GB, CPU 동작 가능 — GPU 불필요)
- 브라우저 접속: http://localhost:8000/frontend/

## 알려진 한계

1. **동시 사용자 1명** — 세이브 파일 하나 + 전역 캐시 구조 (싱글 플레이 전용)
2. **DB 없음** — JSON 파일 저장 (로컬 플레이 용도, 고트래픽 부적합)
3. **인증 없음** — 개인/개발 용도
4. **응답 속도는 하드웨어 의존** — CPU 전용 환경에서 전체 생성에 수 초
   (스트리밍으로 체감은 개선됨)
5. **소형 모델의 한국어 품질** — gemma2:2b는 가끔 어색한 문장이나 영어 혼입
   (`OLLAMA_MODEL`로 큰 모델 교체 가능)
6. **저장 슬롯 없음** — 단일 세이브 (새 게임 시 덮어씀)
7. **스킬/능력 시스템 없음** — 전투는 공격/도주/물약 중심
8. **NPC 대화 시스템 없음** — 인물은 AI 서사 안에서만 등장
